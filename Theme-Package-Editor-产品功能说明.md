# Theme Package Editor 产品功能说明

## 1. 产品概述

Theme Package Editor 是一个运行在本地浏览器中的手机主题包编辑工具，用于打开、浏览、修改并重新导出主题资源包。当前前端源码位于 `frontend/` 目录，拆分为 `index.html`、`styles.css`、`app.js`，已具备完整的单页工具核心编辑能力。

产品核心定位如下：

- 面向主题包资源的轻量级本地编辑工具。
- 支持 `.itz`、`.mtz`、`.zip` 等基于 ZIP 结构的主题包文件。
- 用户无需安装桌面客户端，也无需上传到服务器，即可在浏览器内完成编辑。
- 编辑完成后可重新打包导出，并保持原始文件扩展名。

## 2. 目标用户

- 手机主题设计师
- 主题包资源维护人员
- 需要快速替换图片、修改配置文本的运营或设计人员
- 需要查看主题包目录结构与素材分布的测试/审核人员

## 3. 典型使用场景

- 修改主题包中的 `xml`、`json`、`txt` 等配置文件
- 替换图标、壁纸、预览图等图片资源
- 新增说明文件、配置文件或素材目录
- 批量删除一组不再使用的图片资源
- 对主题包进行结构检查后重新导出测试版本

## 4. 产品形态与使用边界

### 4.1 产品形态

- 单 HTML 页面工具
- 本地浏览器运行
- 前端本地解析与导出，不依赖在线服务

### 4.2 使用边界

- 当前仅支持打开“ZIP 结构”的主题包文件
- 不支持账号、云端存储、多人协作
- 不支持版本比对、撤销/重做、操作历史
- 不支持直接编辑二进制内容，仅支持替换二进制文件

## 5. 功能模块说明

### 5.1 主题包导入

#### 功能说明

用户可通过“打开主题包”按钮选择本地文件导入工具。

#### 支持格式

- `.itz`
- `.mtz`
- `.zip`
- 其他扩展名但实际内容为 ZIP 结构的文件

#### 处理逻辑

- 读取用户选择的本地文件
- 以 ZIP 方式解析主题包内容
- 解析成功后生成左侧目录树
- 解析失败时提示“打开失败”，说明文件可能不是 ZIP 结构或已损坏

#### 结果表现

- 顶部显示当前已打开文件名
- 左侧显示主题结构树
- 右侧显示“已加载，可开始编辑”
- “导出主题包”“清空”等操作按钮显示

### 5.2 主题结构浏览

#### 功能说明

系统将主题包内容解析为树状结构，支持目录浏览、展开收起和文件检索。

#### 具体能力

- 目录与文件按树状结构展示
- 文件夹优先显示，随后按名称排序
- 支持单个目录展开/收起
- 支持“全部展开”“全部收起”
- 支持关键字搜索
- 搜索范围覆盖文件名及路径
- 已修改项目显示“新”标记
- 内部 ZIP 类资源会以特殊目录形式展示

#### 交互规则

- 单击节点可选中项目
- 双击文件夹可展开或收起
- 右侧区域随当前选中项联动展示详情

### 5.3 文件预览与编辑

#### 5.3.1 文件夹预览

选中文件夹时，右侧显示：

- 文件夹名称
- 当前包含的可见项目数量
- 若该目录来自“内部 ZIP 类资源”，则提示其为内部压缩包，导出时自动回写

#### 5.3.2 图片文件预览

系统可识别常见图片资源并提供预览能力。

支持的图片类型包括：

- `png`
- `jpg`
- `jpeg`
- `webp`
- `gif`
- `bmp`
- `svg`

选中图片后，右侧显示：

- 图片预览
- 文件容量
- 图片尺寸（宽 × 高）
- 替换提示

用户可通过“替换文件”按钮，用本地文件覆盖当前图片资源。

#### 5.3.3 文本文件编辑

系统可识别常见文本资源，并在右侧提供文本编辑器。

优先识别的文本扩展名包括：

- `txt`
- `xml`
- `json`
- `ini`
- `cfg`
- `conf`
- `css`
- `html`
- `js`
- `md`
- `csv`
- `properties`
- `theme`
- `manifest`
- `plist`

除扩展名判断外，系统还会对部分未知扩展名内容做文本特征识别，尽量以文本方式打开可读内容。

文本编辑能力包括：

- 展示完整文本内容
- 允许直接修改
- 点击“保存文本”后写回当前文件
- 保存后标记为已修改

当前文本保存采用 UTF-8 编码重新写入。

#### 5.3.4 二进制文件处理

对于既不是图片、也不适合按文本打开的文件，右侧显示二进制文件提示。

该类文件当前支持：

- 查看文件名
- 使用“替换文件”进行整体替换

不支持直接十六进制或字节级编辑。

### 5.4 资源新增

#### 5.4.1 新增文本文件

用户在选中文件夹或文件后，可通过“新增文本”创建新的空文本文件。

规则如下：

- 默认创建位置为当前选中目录
- 若当前选中的是文件，则默认创建在该文件所在目录
- 用户通过弹窗输入目标路径
- 支持输入多级路径，例如 `icons/config.json`
- 若目标文件已存在，可选择是否覆盖
- 若同路径已存在文件夹，则创建失败并提示冲突

创建成功后：

- 文件自动加入目录树
- 自动选中新建文件
- 文件标记为已修改

#### 5.4.2 新增文件夹

用户可通过“新增文件夹”创建目录。

规则如下：

- 支持输入多级目录路径
- 若目录不存在则自动递归创建
- 若同路径已存在文件，则提示冲突
- 若目录已存在，可确认继续使用该目录

创建成功后会自动选中新建或已有目录。

#### 5.4.3 新增文件

用户可通过“新增文件”一次选择一个或多个本地文件导入到当前目录。

规则如下：

- 默认导入到当前选中目录
- 若当前选中的是文件，则导入到其父目录
- 文件保持本地原文件名
- 支持多文件批量导入
- 若存在同名文件，会统一进行覆盖确认
- 若同路径已存在文件夹，则中止导入并提示冲突

### 5.5 文件与目录管理

#### 5.5.1 重命名

用户可对单个文件或文件夹执行重命名。

规则如下：

- 根节点不允许重命名
- 多选状态下不允许重命名
- 新名称不能为空
- 名称中不能包含 `/`
- 同级目录下不能与现有项目重名
- 文件夹重命名后，其所有子路径会同步更新
- 内部压缩包目录重命名后，对应归档资源路径同步更新

#### 5.5.2 单项删除

用户可删除当前选中的单个文件或文件夹。

规则如下：

- 根节点不允许删除
- 删除前弹出确认框
- 删除后返回父级目录视图

#### 5.5.3 批量删除

用户可通过多选模式批量删除素材。

多选规则如下：

- `Shift`：连续选择
- Windows / Linux 使用 `Ctrl` 点选
- macOS 使用 `Command` 点选

批量删除能力如下：

- 多选后右侧切换为素材列表视图
- 列表显示素材名称、文件容量、图片尺寸
- 图片项会尝试读取宽高
- 点击“删除所选”后统一确认
- 若父级目录与子文件同时被选中，删除时会自动去重，避免重复处理

### 5.6 搜索与定位

用户可在左侧搜索框输入关键字快速筛选树节点。

特性如下：

- 支持按文件名搜索
- 支持按完整路径搜索
- 若子节点命中，父目录会保留显示
- 搜索结果随输入实时刷新

### 5.7 导出主题包

#### 功能说明

用户完成修改后，可通过“导出主题包”重新打包下载。

#### 导出规则

- 以当前树结构为准重新生成压缩包
- 已删除项目不参与导出
- 已修改、新增、替换内容全部写入导出包
- 导出文件名格式为：`原文件名_modified + 原扩展名`
- 例如：`theme.itz` 导出为 `theme_modified.itz`

#### 内部压缩包处理

当主题包内存在“实际内容为 ZIP”的内部资源时，系统会：

- 在编辑时按目录方式展开显示
- 导出时重新打包为原始压缩资源
- 保持其原有文件名与所在路径

该能力适合处理某些主题包中嵌套的资源容器文件。

### 5.8 当前包清空

用户可点击“清空”关闭当前已打开主题包。

执行后：

- 清除当前加载状态
- 隐藏编辑相关操作按钮
- 恢复初始使用说明页面
- 清空搜索与选择状态

### 5.9 主题模式切换

页面右下角提供主题切换按钮，支持浅色/深色模式切换。

特性如下：

- 手动切换浅色与深色界面
- 用户选择会保存在浏览器本地
- 下次打开页面时沿用上次主题设置

## 6. 页面结构说明

### 6.1 左侧区域

- 产品标题与说明
- 打开主题包
- 导出主题包
- 清空当前包
- 主题结构树
- 搜索框
- 展开/收起入口
- 多选操作说明

### 6.2 右侧区域

- 当前路径显示
- 当前项目元信息显示
- 文件操作按钮区
- 文件夹/图片/文本/二进制预览区

### 6.3 弹窗能力

统一用于以下场景：

- 提示
- 确认
- 覆盖确认
- 删除确认
- 输入文件路径
- 输入文件夹路径
- 输入新名称

## 7. 业务规则与限制说明

- 仅支持本地文件选择打开，当前页面未实现拖拽导入
- 工具默认面向 ZIP 结构主题包，不保证支持非 ZIP 格式容器
- 文本编辑需用户主动点击“保存文本”，未保存内容不会自动写回
- 当前未提供“保存单文件到本地”的独立导出能力，仅支持整包导出
- 当前未提供撤销、重做、差异对比、批量替换规则等高级功能
- 多选状态下主要支持批量查看与批量删除，不支持批量重命名或批量替换
- 文件删除为当前编辑会话内删除，最终是否生效以导出结果为准
- 文件路径规范使用 `/` 作为分隔符

## 8. 建议的对外产品卖点

- 本地浏览器运行，使用门槛低
- 无需解包再手工回包，缩短主题修改流程
- 目录树浏览清晰，适合快速定位资源
- 文本可直接改、图片可直接换，操作直观
- 支持批量删除和路径级新增，适合主题包整理
- 可处理部分嵌套压缩资源，减少特殊主题包处理成本

## 9. 总结

`frontend/` 当前已经实现了一个可独立使用的主题包编辑页面，覆盖了“打开主题包、浏览结构、搜索定位、编辑文本、替换图片/二进制、新增文件/文件夹、重命名、删除、批量删除、重新导出”的完整核心链路，适合作为主题包轻量编辑工具的首版产品能力说明。
