# Theme Package Editor 产品功能说明

## 1. 产品概述

Theme Package Editor 是一个面向手机主题包资源的本地编辑工具，既可作为浏览器页面使用，也已支持通过 Tauri 打包为桌面应用。

产品核心定位如下：

- 面向主题包资源的轻量级本地编辑工具。
- 支持 `.itz`、`.mtz`、`.zip` 等 ZIP 结构主题包。
- 支持直接导入普通文件夹，适合从主题工程目录开始制作。
- 文本、图片、目录结构均可在本地完成修改，无需上传到服务器。
- 支持“保存到当前来源”和“导出为新主题包”两种结果处理方式。

## 2. 目标用户

- 手机主题设计师
- 主题包资源维护人员
- 需要快速替换图片、修改配置文本的运营或设计人员
- 需要查看主题包目录结构与素材分布的测试或审核人员

## 3. 典型使用场景

- 修改主题包中的 `xml`、`json`、`txt`、`css`、`html`、`js` 等配置文件
- 替换图标、壁纸、预览图等图片资源
- 新增说明文件、配置文件或素材目录
- 批量删除一组不再使用的图片资源
- 从普通文件夹导入主题工程并继续编辑
- 编辑完成后直接保存回当前来源，或导出为测试版本

## 4. 产品形态与使用边界

### 4.1 产品形态

- 浏览器页面工具
- Tauri 桌面应用
- 前端本地解析、编辑、导出，不依赖在线服务

### 4.2 使用边界

- 当前主要面向 ZIP 结构主题包和普通文件夹工程
- 不支持账号、云端存储、多人协作
- 不支持版本比对、撤销、重做
- 不支持直接编辑二进制字节内容，仅支持替换整个文件
- 浏览器环境下是否支持“直接保存回原位置”，取决于浏览器是否支持 File System Access API

## 5. 功能模块说明

### 5.1 导入主题内容

#### 功能说明

用户可通过左侧 `导入` 按钮展开导入菜单，选择导入方式。

#### 导入方式

- 导入主题包
- 导入文件夹

#### 主题包导入支持格式

- `.itz`
- `.mtz`
- `.zip`
- 其他扩展名但实际内容为 ZIP 结构的文件

#### 文件夹导入说明

- 支持导入普通本地文件夹作为主题工程
- 导入后按目录树形式编辑
- 适合尚未打包、仍以工程目录维护资源的用户

#### 导入前提醒

当当前主题内容存在未保存或未导出的修改时，再次导入新内容前会弹出提醒：

- 文案提示建议先“保存或导出”当前主题内容
- 用户可选择 `返回` 或 `继续导入`
- `返回` 会关闭弹窗并停留在当前工程

#### 处理逻辑

- 读取用户选择的本地文件或文件夹
- ZIP 文件按压缩包方式解析
- 文件夹按目录结构构造成主题树
- 解析成功后生成左侧目录树
- 解析失败时提示“打开失败”

#### 结果表现

- 左侧显示主题结构树
- 右侧进入可编辑状态
- 树底部显示 `保存`、`导出` 操作栏
- 若当前环境支持，将记录导入来源，用于后续直接保存回原位置

### 5.2 主题结构浏览

#### 功能说明

系统将主题内容解析为树状结构，支持目录浏览、展开收起和文件检索。

#### 具体能力

- 目录与文件按树状结构展示
- 文件夹优先显示，随后按名称排序
- 支持单个目录展开或收起
- 支持 `全部展开`、`全部收起`
- 支持关键字搜索
- 搜索范围覆盖文件名及路径
- 已修改项目显示标记
- 内部 ZIP 类资源会以特殊目录形式展示

#### 交互规则

- 单击节点可选中项目
- 双击文件夹可展开或收起
- 右侧区域随当前选中项联动展示详情
- 支持多选并批量删除

### 5.3 文件预览与编辑

#### 5.3.1 文件夹预览

选中文件夹时，右侧显示：

- 文件夹名称
- 当前包含的可见项目数量
- 若该目录来自内部 ZIP 类资源，则提示其为内部压缩包，导出或保存时自动回写

#### 5.3.2 图片文件预览

系统可识别常见图片资源并提供预览能力。

支持的图片类型包括：

- `png`
- `jpg`
- `jpeg`
- `webp`
- `gif`
- `bmp`
- `svg`

选中图片后，右侧显示：

- 图片预览
- 文件容量
- 图片尺寸
- 替换提示

用户可通过 `替换文件` 按钮，用本地文件覆盖当前图片资源。

#### 5.3.3 文本文件编辑

系统可识别常见文本资源，并在右侧提供代码编辑器。

优先识别的文本扩展名包括：

- `txt`
- `xml`
- `json`
- `ini`
- `cfg`
- `conf`
- `css`
- `html`
- `js`
- `md`
- `csv`
- `properties`
- `theme`
- `manifest`
- `plist`

除扩展名判断外，系统还会对部分未知扩展名内容做文本特征识别，尽量以文本方式打开可读内容。

#### 编辑能力

- 展示完整文本内容
- 允许直接修改
- 点击 `保存文本` 后写回当前文件节点
- 保存后标记为已修改
- 使用 UTF-8 编码重新写入

#### 编辑器增强能力

- 针对 `XML / HTML / CSS / JS / JSON` 提供格式化对齐
- `XML / HTML / CSS / JS / Markdown` 支持 `Ctrl/Cmd + /` 注释或取消注释
- `XML / PLIST` 支持更适合结构文本的格式化缩进
- `XML` 提供标签、属性补全与匹配能力
- 不同类型文本会自动加载对应 CodeMirror 模式

#### 未保存保护

- 编辑器内容被修改后，`保存文本` 按钮会进入未保存状态
- 若当前文本尚未保存，切换文件前会弹出提醒
- 提醒提供：
  - `保存`
  - `不保存`
  - `取消`
- 该提醒只处理当前文件，不会自动跳转到目标文件

#### 5.3.4 二进制文件处理

对于既不是图片、也不适合按文本打开的文件，右侧显示二进制文件提示。

该类文件当前支持：

- 查看文件名
- 使用 `替换文件` 进行整体替换

不支持直接十六进制或字节级编辑。

### 5.4 资源新增

#### 5.4.1 新增文本文件

用户在选中文件夹或文件后，可通过 `新增文本` 创建新的空文本文件。

规则如下：

- 默认创建位置为当前选中目录
- 若当前选中的是文件，则默认创建在该文件所在目录
- 用户通过弹窗输入目标路径
- 支持输入多级路径，例如 `icons/config.json`
- 若目标文件已存在，可选择是否覆盖
- 若同路径已存在文件夹，则创建失败并提示冲突

创建成功后：

- 文件自动加入目录树
- 自动选中新建文件
- 文件标记为已修改

#### 5.4.2 新增文件夹

用户可通过 `新增文件夹` 创建目录。

规则如下：

- 支持输入多级目录路径
- 若目录不存在则自动递归创建
- 若同路径已存在文件，则提示冲突
- 若目录已存在，可确认继续使用该目录

创建成功后会自动选中新建或已有目录。

#### 5.4.3 新增文件

用户可通过 `新增文件` 一次选择一个或多个本地文件导入到当前目录。

规则如下：

- 默认导入到当前选中目录
- 若当前选中的是文件，则导入到其父目录
- 文件保持本地原文件名
- 支持多文件批量导入
- 若存在同名文件，会统一进行覆盖确认
- 若同路径已存在文件夹，则中止导入并提示冲突

### 5.5 文件与目录管理

#### 5.5.1 重命名

用户可对单个文件或文件夹执行重命名。

规则如下：

- 根节点不允许重命名
- 多选状态下不允许重命名
- 新名称不能为空
- 名称中不能包含 `/`
- 同级目录下不能与现有项目重名
- 文件夹重命名后，其所有子路径会同步更新
- 内部压缩包目录重命名后，对应归档资源路径同步更新

#### 5.5.2 单项删除

用户可删除当前选中的单个文件或文件夹。

规则如下：

- 根节点不允许删除
- 删除前弹出确认框
- 删除后返回父级目录视图

#### 5.5.3 批量删除

用户可通过多选模式批量删除素材。

多选规则如下：

- `Shift`：连续选择
- Windows / Linux 使用 `Ctrl` 点选
- macOS 使用 `Command` 点选

批量删除能力如下：

- 多选后右侧切换为素材列表视图
- 列表显示素材名称、文件容量、图片尺寸
- 图片项会尝试读取宽高
- 点击 `删除所选` 后统一确认
- 若父级目录与子文件同时被选中，删除时会自动去重，避免重复处理

### 5.6 搜索与定位

用户可在左侧搜索框输入关键字快速筛选树节点。

特性如下：

- 支持按文件名搜索
- 支持按完整路径搜索
- 若子节点命中，父目录会保留显示
- 搜索结果随输入实时刷新

### 5.7 保存与导出

#### 5.7.1 保存

`保存` 用于将当前工程直接写回到当前导入来源。

适用场景：

- 当前导入的是主题包文件
- 当前导入的是主题文件夹
- 当前环境具备原位置写回能力

保存逻辑如下：

- 若当前文本编辑器仍有未保存内容，会先写回当前文件节点
- 然后再将整个工程保存回原文件或原文件夹
- 保存成功后清除当前工程的未保存状态

#### 5.7.2 导出

`导出` 用于将当前工程重新打包为新的主题包文件。

导出规则如下：

- 以当前树结构为准重新生成压缩包
- 已删除项目不参与导出
- 已修改、新增、替换内容全部写入导出包
- 导出文件名格式为：`原文件名_modified + 原扩展名`
- 例如：`theme.itz` 导出为 `theme_modified.itz`

#### 5.7.3 内部压缩包处理

当主题内容内存在“实际内容为 ZIP”的内部资源时，系统会：

- 在编辑时按目录方式展开显示
- 导出或保存时重新打包为原始压缩资源
- 保持其原有文件名与所在路径

#### 5.7.4 最近操作记录

树底部 `保存 / 导出` 操作栏上方会显示最近一次成功操作记录，例如：

- `16:00:01 已保存`
- `16:00:04 已导出`

记录会在导入新内容或清空当前主题后重置。

#### 5.7.5 成功结果展示

保存完成或导出完成后，右侧区域会显示统一成功态：

- 完成图标
- `保存完成` 或 `导出完成`
- 实际保存或导出路径

三项内容居中分行展示。

### 5.8 当前包清空

用户可点击 `清空` 关闭当前已导入的主题内容。

执行后：

- 清除当前加载状态
- 隐藏编辑相关操作按钮
- 恢复初始使用说明页面
- 清空搜索与选择状态

### 5.9 主题模式切换

页面右下角提供主题切换按钮，支持浅色或深色模式切换。

特性如下：

- 手动切换浅色与深色界面
- 用户选择会保存在本地
- 下次打开页面时沿用上次主题设置

### 5.10 操作反馈

除弹窗与右侧结果展示外，系统还提供轻量 toast 反馈。

当前已覆盖的操作包括：

- 单项删除
- 批量删除
- 替换文件
- 新增文本文件
- 新增文件夹
- 新增文件
- 重命名

toast 用于提示“操作已成功完成”，不承担错误说明。

## 7. 业务规则与限制说明

- 工具主要面向 ZIP 结构主题包与普通文件夹工程
- 文本编辑需用户主动点击 `保存文本`，未保存内容不会自动写回当前文件节点
- 浏览器环境下是否可直接 `保存` 回原位置，取决于浏览器是否支持 File System Access API
- 不支持撤销、重做、差异对比、批量替换规则等高级功能
- 多选状态下主要支持批量查看与批量删除，不支持批量重命名或批量替换
- 文件删除既可能通过 `导出` 生效，也可能通过 `保存` 直接回写到当前来源
- 文件路径规范使用 `/` 作为分隔符

## 8. 建议的对外产品卖点

- 本地运行，使用门槛低
- 同时支持主题包导入与文件夹导入
- 无需解包再手工回包，缩短主题修改流程
- 文本可直接改、图片可直接换，目录可直接整理
- 提供保存回原位置与导出新包两种结果路径
- 目录树浏览清晰，适合快速定位资源
- 支持批量删除和路径级新增，适合主题包整理
- 可处理部分嵌套压缩资源，减少特殊主题包处理成本

## 9. 总结

Theme Package Editor 当前已经覆盖了“导入主题包/文件夹、浏览结构、搜索定位、编辑文本、格式化、注释、替换图片与二进制、新增文件/文件夹、重命名、删除、批量删除、保存回当前来源、导出新主题包、操作反馈”等完整核心链路，可作为主题包轻量编辑工具的当前版本功能说明。
